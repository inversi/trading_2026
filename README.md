# trading_2026

Коротко о ключевых настройках и защите:
- `HARD_STOP_LOSS_PCT` — жёсткий стоп по просадке от цены входа (по умолчанию 0.15). Можно переопределить в `.env`. Применяется к учтённым позициям и к балансам, обнаруженным вручную/«пыли».
- `VERBOSE_CTX` — отдельный флаг для подробных логов контекста сигналов (`входа нет — ctx=...`). По умолчанию `false`, чтобы не засорять `logs/app.log`.
- Dust/ручные остатки: при инициализации теперь не отбрасываются остатки ниже minNotional. В `on_tick` выполняется принудительный выход при просадке ≥ `HARD_STOP_LOSS_PCT`, пытаясь продать весь доступный объём с проверкой minNotional/precision.

Рекомендуется:
1) Проверять изменения в `MODE=paper` перед `live`.
2) Явно задавать `HARD_STOP_LOSS_PCT` и другие рисковые параметры в `.env`.
3) Включать `VERBOSE_CTX=true` только для отладки.
